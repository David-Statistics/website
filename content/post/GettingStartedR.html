What is R?

R is an open source statistical programming language. It is completely free to use, offers very powerful functionality, and the ability to code in R can be a very valuable skill on your resume and in many jobs.

How to Download R and RStudio

Your computer may actually come with R installed, but just to ensure it’s up to date, go ahead and go through the process anyway. As of writing this (2/22/2018), the most recent version is 3.4.3 “Kite-Eating Tree.” The process for getting installing R is quite simple, go to https://cloud.r-project.org/ and select “Download for [your operating system]” (on Windows you’ll want base). Once that is installed, go to https://www.rstudio.com/products/rstudio/#Desktop and download and install the open source (free) version of RStudio desktop.

Note this post in particular doesn’t ask you to do much - you should spend some time just exploring the RStudio IDE - don’t just do the same things that are shown below.

Using RStudio

RStudio is an integrated development environment (IDE) for the R language. You should see either 3 or 4 panels when you open RStudio for the first time. If you see only 3 panels (1 on the left and 2 on the right) go ahead and click the blank page icon in the upper left corner and select ‘R script’ to get the 4th panel. The upper left panel is both your data viewer as well as your text editor. It will be where you do the vast majority of your typing. The lower left panel is the console and it is where code is actually run. You can run code line by line directly from the console or you can highlight text in the upper left panel and run the highlighted code by pressing command-enter on Macs or control-enter on Windows (it’s recommended that you get used to the latter so that your code is saved and reproducible). The upper right panel has several different options but is usually going to show your current environment - that is, a list of objects which you have created - it should be empty upon opening RStudio for the first time. The lower right panel again has several options but will usually used as either a file navigator or as a a plot viewer.

Using R

Throughout these exercises, I’ll include R code that you should try running. Let’s get started by using R as a simple calculator. Try typing the following into an R script, then highlighting it and running it (control or command enter):

2 + 2

You should see

## [1] 4

in your console.

The basic math operators +,-,*,/ all work as you think they should in R. R also knows the order of operations so we get the correct result for the following,

3 + 2 * 4
## [1] 11

As you’re just getting used to R, use the examples as a guideline for the types of things you can do. I would expect you to play around with R more than simply copying and pasting the exact expressions I give you - especially if you’ve never worked with a programming language before.

Variables and Assignment

If we were to only use R as a calculator it would be like owning and using a car for the sole purpose of driving to our next door neighbor’s house. Many times we want to store results so that we can use them at a later point without having to do the calculation over again. We do that by using <- or = (for these exercises, I will generally use the former) in between the name we want to give to the result and the expression which gives us the result. For instance,

x <- 2 + 2 * 2

Now, we can call x or do further operations with x,

x
## [1] 6
x * 2
## [1] 12

Most of the variables we’ll be working with at first will be numeric variables or vectors (next section), however there are more variable types which will come up later (such as strings and factors). For instance,

x <- "Hello World"

x
## [1] "Hello World"

Vectors

A vector in R is one of the most common data stuctures that we’ll work with. Simply put, a vector is merely an ordered collection of values which are all of the same type. We make a vector by using the c() function. For instance, if we want to make a vector of the integers between 1 and 5 and give it the name ints we would run

ints <- c(1, 2, 3, 4, 5)

Now when we call this object, this is what happens,

ints
## [1] 1 2 3 4 5

Again, most vectors we work with will be numeric variables but this doesn’t have to be the case. For example,

ABCs = c("a", "b", "c", "d", "e", "f", "g", "h", "i", "j", "k", "l", "m", 
         "n", "o", "p", "q", "r", "s", "t", "u", "v", "w", "x", "y", "z")

ABCs
##  [1] "a" "b" "c" "d" "e" "f" "g" "h" "i" "j" "k" "l" "m" "n" "o" "p" "q"
## [18] "r" "s" "t" "u" "v" "w" "x" "y" "z"

Operations

Most operations in R are what we call vectorized. That means that the operation that you’re trying to do will be applied to each element of a vector and if it’s an operation across two vectors, it will be applied element-wise. It’s generally easier to see what this means through example though:

x = c(0, 1, 2)
y = c(3, 4, 5)
2 * x
## [1] 0 2 4
y + 2
## [1] 5 6 7
x + y
## [1] 3 5 7
x * y
## [1]  0  4 10
exp(x)
## [1] 1.000000 2.718282 7.389056
log(y)
## [1] 1.098612 1.386294 1.609438

Help pages

Not sure exactly what the exp function above actually does? Not sure what base log uses? Type the following into the console,

?exp
?log
